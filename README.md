# Finance Tracker

## Описание

Серверная часть приложения финансового трекера.

## Предпосылки

Мотивацией проекта служит желание автора создать веб-приложение без использования набора инструментов Spring и контейнера приложений Apache Tomcat.

## Технологическая среда

Среда выполнения содержит пакеты: 

- Java SE 17;
- сборщик пакетов Apache Maven.

Все команды запускаются в консоли из папки проекта ``FinanceTracker/``.

Первоначальные настройки отсутствуют.

## Сборка

Сборка проекта производится командой 
```
mvn clean package
``` 
При успешном завершении результаты сборки будут расположены в папке ``target/``

Для возможности запуска приложения из контейнера необходимо построить образ контейнера на основе образа, содержащего Java JDK 17, при помощи следующей команды:
```
docker build --tag finance-tracker .
```

Сборканеобходимых контейнеров из созданных образов производится коммандой:
```
docker compose --project-directory ./docker/ up --detach
```

## Запуск

Для запуска приложения в контейнере необходимо выполнить следующие действия.

Поскольку приложение помимо клиент-серверного взаимодействия через HTTP-запросы предполагает диалог с пользователем через меню в псевдо-терминале, для запуска приложения могут быть использованы следующие варианты команд:
1. без интерактивного меню:
```
docker compose --project-directory ./docker/ run --detach --rm app
```
2. с интерактивным меню:
```
docker compose --project-directory ./docker/ run --rm app
```

## Инструкции по использованию

Использование функциональности приложения в режиме клиент-серверного взаимодействия производится посредством перехода по сетевой ссылке `http://localhost:8088/<endpoint>`, где `<endpoint>` представляет отдельную API команду.
Приложение использует базовую аутентификацию, при этом в качестве ``Username`` указывается адрес электронной почты пользователя.

Приложение подразумевает выполнение следующих API команд:

### Команды сервиса <span style="color: lightgreen">${auth}$</span>

- <span style="color: lightblue">${/api/v1/auth/create}$</span> - вызов команды производится с помощью HTTP метода `POST`. Команда ожидает входные данные в виде объекта в `JSON` нотации `{ "email": <адрес электронной почты>, "password": <пароль> }`. Результатом успешного выполнения данной команды является создание новой учетной записи;

### Команды сервиса <span style="color: lightgreen">${identity}$</span>

- <span style="color: lightblue">${/api/v1/identity/delete}$</span> - вызов команды производится с помощью HTTP метода `DELETE`. Результатом успешного выполнения данной команды является удаление учетной записи пользователя и завершение текущей сессии;<br><br>
- <span style="color: lightblue">${/api/v1/identity/update}$</span> - вызов команды производится с помощью HTTP метода `PUT`. Команда ожидает входные данные в виде объекта в `JSON` нотации `{ "name": <имя>, """email": <адрес электронной почты>, "password": <пароль> }`. Результатом успешного выполнения данной команды является обновление пароля учетной записи пользователя текущей сессии;

### Команды сервиса <span style="color: lightgreen">${administration}$</span>

- <span style="color: lightblue">${/api/v1/administration/list}$</span> - вызов команды производится с помощью HTTP метода `GET`. Команда ожидает входные данные в виде объекта в `JSON` нотации `{ "name": <имя>, "email": <адрес электронной почты>, "password": <пароль>, "role": <ADMIN|USER>, "status": <ACTIVE|BLOCKED> }`. Результатом успешного выполнения данной команды является возврат перечня пользователей, удовлетворяющего критериям указанным в виде `JSON` нотации;<br><br>
- <span style="color: lightblue">${/api/v1/administration/transactions}$</span> - вызов команды производится с помощью HTTP метода `GET`. Команда ожидает в HTTP-запросе целочисленного значения для параметра `id`. Результатом успешного выполнения данной команды является возврат перечня транзакций пользователя, удовлетворяющего пользователю с указанным идентификатором;<br><br>
- <span style="color: lightblue">${/api/v1/identity/delete}$</span> - вызов команды производится с помощью HTTP метода `DELETE`. Команда в HTTP-запросе ожидает целочисленного значения для параметра `id`. Результатом успешного выполнения данной команды является удаление учетной записи с указанным идентификатором;<br><br>
- <span style="color: lightblue">${/api/v1/identity/block}$</span> - вызов команды производится с помощью HTTP метода `DELETE`. Команда в HTTP-запросе ожидает целочисленного значения для параметра `id`. Результатом успешного выполнения данной команды является установка статуса `BLOCKED` для учетной записи с указанным идентификатором;<br><br>

### Команды сервиса <span style="color: lightgreen">${transaction}$</span>

- <span style="color: lightblue">${/api/v1/transaction/create}$</span> - вызов команды производится с помощью HTTP метода `POST`. Команда ожидает входные данные в виде объекта в `JSON` нотации `{ "type": <тип>, "category": <категория>, "amount": <сумма>, "description": <описание> }`. Результатом успешного выполнения данной команды является создание новой записи транзакции;<br><br>
- <span style="color: lightblue">${/api/v1/transaction/list}$</span> - вызов команды производится с помощью HTTP метода `GET`. Команда ожидает входные данные в виде объекта в `JSON` нотации `{ "type": <тип>, "category": <категория>, "amount": <сумма>, "description": <описание> }`. Результатом успешного выполнения данной команды является возврат перечня транзакций, соответствующих шаблону указанному в теле запроса.<br><br>
- <span style="color: lightblue">${/api/v1/transaction/update}$</span> - вызов команды производится с помощью HTTP метода `PUT`. Команда ожидает входные данные в виде объекта в `JSON` нотации `{ "date": <YYYY-MM-DD T HH::mm::ssZ>, "type": <тип>, "category": <категория>, "amount": <сумма>, "description": <описание> }`. Результатом успешного выполнения данной команды является обновление записи транзакции, исходя из момента её проведения;<br><br>
- <span style="color: lightblue">${/api/v1/transaction/delete}$</span> - вызов команды производится с помощью HTTP метода `DELETE`. Команда ожидает входные данные в виде объекта в `JSON` нотации `{ "date": <YYYY-MM-DD T HH::mm::ssZ> }`. Результатом успешного выполнения данной команды является удаление записи транзакции, исходя из момента её проведения;

### Команды сервиса <span style="color: lightgreen">${limit}$</span>

Результатом успешного выполнения данной команды является подтверждения установки/отключения уведомлений в случае проведения транзакций с расходами на сумму превышающую установленный лимита финансирования;

### Команды сервиса <span style="color: lightgreen">${analytics}$</span>

- <span style="color: lightblue">${/api/v1/analytics/balance}$</span> - вызов команды производится с помощью HTTP метода `GET`. Результатом успешного выполнения данной команды является возврат информации о балансе счета для текущего пользователя;<br><br>
- <span style="color: lightblue">${/api/v1/analytics/summary}$</span> - вызов команды производится с помощью HTTP метода `GET`. Результатом успешного выполнения данной команды является возврат информации о суммарных расходах и суммарных доходах текущего пользователя, за указанный период.<br><br>

- <span style="color: lightblue">${/api/v1/fund/condition}$</span> - вызов команды производится с помощью HTTP метода `GET`. Результатом успешного выполнения данной команды является информация о неизрасходованном лимите финансирования на текущий месяц и сумме накоплений во внебюджетных фондах для текущего пользователя;
